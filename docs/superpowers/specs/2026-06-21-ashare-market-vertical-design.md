# A 股全维度分析 Vertical — 设计文档

> 基于 Brainstorming 结果，2026-06-21

## 概述

为 Sulix Intelligence 新增 **A 股（A-share）** 分类，将 Red-Blue 对抗分析框架引入中国二级市场信息流。目标是在信息极度高频、噪音极大、情绪易被小作文裹挟的 A 股市场中，提供一个冷静、客观、带蓝军批判视角的自动化分析工具。

## 分析框架

### 六维交叉比对（在原五维上增加"背离检测"为独立维度）

| 维度 | 红军视角（机会） | 蓝军视角（风险） |
|------|----------------|----------------|
| **资金面** | 主力资金布局方向、北向资金趋势 | 单日数据噪音、真假外资、拉高出货 |
| **技术面** | 换手率结构、成交量、技术指标共振 | 技术分析的自我实现风险、流动性陷阱 |
| **政策面** | 产业政策红利方向、监管放松信号 | 落地力度不足、利好出尽 |
| **基本面** | 低估价值洼地、景气度拐点 | 估值陷阱、景气度下行被忽略 |
| **情绪面** | 市场过冷=买入窗口 | 情绪指标滞后、散户追高 |
| **背离检测**（核心） | — | **跨维度交叉比对**：价格vs资金背离 / 情绪vs资金背离 / 政策vs量价背离 |

### 背离检测：红蓝对抗的银弹

强制 Agent 执行跨维度交叉比对，这是 A 股过滤噪音的核心机制：

- **情绪面 vs 资金面**：电报一片看好、涨停家数新高，但主力资金放量滞涨 → **输出：【高危背离：散户情绪过热，主力资金假掩护】**
- **政策面 vs 技术面**：政策大利好，但板块高开低走，成交量无法持续放大 → **输出：【利好出尽，市场流动性不足，轮动进入末期】**

### 信号质量分级：A 股含金量漏斗（L1-L5）

| 等级 | 标签 | 含义 | 创业者的行动 |
|------|------|------|------------|
| L1 | ❌ 杂音 | 个股盘中波动、常规聚焦、无主线小作文 | 忽略 |
| L2 | ⚠️ 异动 | 板块短线换手率突破、单日主力净流入 | 观察，需交叉验证 |
| L3 | ✅ 共振 | 政策发文 + 主力资金连续 3 日净流入 + 技术形态共振 | 研究，可制定计划 |
| L4 | ✅ 主线趋势 | 行业景气度拐点 + 央行/产业政策强力落地 | 可行动 |
| L5 | 🔥 系统窗口 | 市场冰点期 + 估值历史洼地 + 大周期信号 | 立即行动 |

### 防抖策略

财联社电报的吞吐量是**恐怖级别的（单日数百条）**。必须做两层前置过滤：

**第一层：关键词布尔拦截（在 fetcher.rs 层面）**
在 Rust fetch 阶段用简单正则/关键词列表过滤，**只保留包含以下领域的条目**：
- 大盘宏观：大盘、指数、沪指、创业板、成交量、万亿
- 行业板块：板块、轮动、半导体、AI、算力、新能源、汽车、消费、医药、金融
- 资金信号：主力、北向、融资、融券、净流入、净流出
- 政策信号：政策、监管、央行、证监会、产业基金、补贴
- 财报信号：财报、季报、业绩、预增、暴雷

预计砍掉 60-70% 的噪音条目。

**第二层：批处理时间窗**
不要逐条处理。快讯按 **1-2 小时窗口** 打包，Scan Agent 一次性批处理，提炼《近两小时盘中焦点与资金流向背离报告》。

## 文件改动

### 1. `config.toml`

新增 `[[sources]]` 块（5 个源）和 `[prompts.vertical_overrides]"A股"` 覆盖 prompt。

### 2. `config.example.toml`

同步示例配置，注释说明防抖策略。

### 3. `src/fetcher.rs`（可选）

如果需要在 Rust 层面实现关键词预过滤，在 `fetch_single_source` 中加一个关键词白名单过滤器，只对 `category = "A股"` 的源生效。不做也行——Scan Agent 的 LLM 初筛已经能过滤大部分噪音，但算力成本会高一些。

## 验证

1. `cargo build && cargo test && cargo clippy` — 编译和测试通过
2. 添加 RSSHub 源后手动运行一次管线，确认 A 股分类的文章被正确分组
3. 查看日报输出，确认 override prompt 生效（5 维度 + L1-L5 评级 + 背离检测）
